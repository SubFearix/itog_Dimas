# ВВЕДЕНИЕ

В условиях стремительной цифровизации современного общества, охватывающей как транспортную отрасль, так и сферу информационной безопасности, возрастает потребность в разработке специализированных программных решений, обеспечивающих как функциональность, так и защищенность информационных систем. Данная курсовая работа посвящена комплексному исследованию двух взаимодополняющих направлений современной разработки программного обеспечения: созданию защищенных клиент-серверных приложений и разработке образовательных платформ для изучения уязвимостей информационных систем.

## Актуальность работы

Актуальность исследования обусловлена несколькими ключевыми факторами:

1. **Рост киберугроз и необходимость подготовки специалистов.** В условиях постоянного роста числа киберинцидентов и усложнения методов атак подготовка квалифицированных специалистов в области информационной безопасности становится критически важной задачей. Практические соревнования CTF (Capture The Flag) зарекомендовали себя как один из наиболее эффективных методов обучения и оценки навыков специалистов по информационной безопасности.

2. **Цифровизация транспортной отрасли.** Системы автоматизации бронирования билетов становятся критически важным связующим звеном между перевозчиком и пассажиром. Переход к онлайн-сервисам требует не только высокой скорости обработки данных, но и обеспечения максимального уровня безопасности личных и платежных данных пользователей.

3. **Противоречие между функциональностью и безопасностью.** Современные информационные системы должны совмещать удобство использования с высоким уровнем защиты данных, что требует глубокого понимания как принципов построения защищенных приложений, так и методов их потенциального взлома.

## Состояние проблемы

Анализ текущего состояния в рассматриваемых областях выявляет следующие проблемы:

**В области информационной безопасности:**
- Многие уязвимости веб-приложений не могут быть эффективно изучены без практического взаимодействия с уязвимым кодом
- Существует дефицит качественных учебных платформ, моделирующих реалистичные уязвимые среды
- Отсутствие стандартизированных методик проверки и валидации успешных атак в образовательных целях

**В области разработки клиент-серверных систем:**
- Современные системы бронирования часто либо слишком громоздки, либо недостаточно защищены на этапе верификации пользователя
- Проблемы масштабируемости и безопасности при обработке персональных и платежных данных
- Необходимость защиты информационного взаимодействия между клиентом и сервером от несанкционированного доступа

## Цели и задачи работы

**Основная цель** данной курсовой работы заключается в разработке комплексного решения, объединяющего два направления:

1. Проектирование и реализация уязвимого веб-сервиса «Секретариат» для проведения учебных и соревновательных мероприятий по информационной безопасности с интегрированной системой генерации и проверки флагов.

2. Разработка защищенного клиент-серверного приложения для бронирования железнодорожных билетов с интуитивно понятным графическим интерфейсом и механизмами безопасного обмена данными.

Для достижения поставленных целей необходимо решить следующие **задачи**:

### Задачи по разработке уязвимого веб-сервиса:
1. Провести анализ существующих платформ и методологий проведения CTF-соревнований, а также типовых уязвимостей веб-приложений
2. Спроектировать и реализовать функциональный веб-сервис с ролевой моделью (студент, преподаватель, администратор)
3. Намеренно внедрить в код сервиса набор уязвимостей различных категорий (инъекции, ошибки аутентификации и авторизации, логические ошибки)
4. Разработать механизм генерации, выдачи и проверки флагов (CTF flags) при успешной эксплуатации каждой уязвимости
5. Реализовать систему детального логирования действий пользователей и атак для последующего анализа
6. Обеспечить безопасную изоляцию уязвимого сервиса в отдельном контейнере (Docker) и подготовить сценарии развёртывания

### Задачи по разработке системы бронирования:
1. Изучить принципы построения клиент-серверных архитектур и методы обеспечения сетевой безопасности
2. Спроектировать базу данных для хранения информации о рейсах, пользователях и истории бронирования
3. Разработать клиентское приложение на базе Qt Creator с реализацией системы регистрации, авторизации и верификации через email
4. Реализовать функционал поиска билетов с фильтрацией по маршрутам и датам, а также интерактивную схему выбора мест в вагоне
5. Организовать систему автоматического заполнения данных профиля и генерации отчетных документов (билетов) в формате PDF
6. Настроить серверную часть для корректной обработки запросов и управления транзакциями

## Ожидаемые результаты

**По направлению информационной безопасности:**
Ожидается создание полностью функционирующего уязвимого веб-сервиса, который позволит участникам соревнований отрабатывать навыки поиска и эксплуатации уязвимостей в условиях, приближённых к реальным. Система будет включать автоматизированную проверку флагов и детальное логирование для анализа действий участников.

**По направлению разработки клиент-серверного приложения:**
Будет разработана полнофункциональная система бронирования железнодорожных билетов, позволяющая пользователю пройти полный путь от поиска рейса до получения электронного билета на почту. Система будет характеризоваться механизмом верификации аккаунтов для предотвращения ложных бронирований и модульной архитектурой клиента, обеспечивающей быструю навигацию между профилем, историей поездок и процессом покупки.

## Новизна и практическая значимость

Новизна работы заключается в:
- Комплексном подходе к изучению информационной безопасности через создание реалистичной уязвимой среды с автоматизированной системой валидации
- Реализации механизма верификации аккаунтов через email в системе бронирования для предотвращения ложных бронирований
- Использовании современных технологий контейнеризации (Docker) для обеспечения изоляции уязвимых компонентов
- Применении кроссплатформенного фреймворка Qt для создания производительного клиентского приложения

**Практическая значимость** работы определяется возможностью использования разработанных систем:
- В образовательных учреждениях для проведения практических занятий и соревнований по информационной безопасности
- В качестве базы для создания коммерческих систем бронирования билетов с высоким уровнем защиты данных
- Для демонстрации принципов безопасной разработки программного обеспечения и методов защиты от типовых уязвимостей

Таким образом, данная работа объединяет теоретические знания в области информационной безопасности и разработки клиент-серверных приложений с практическими навыками создания функциональных программных систем, что формирует комплексное понимание современных подходов к разработке защищенного программного обеспечения.
