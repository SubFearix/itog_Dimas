# РАЗДЕЛ 1. ЗАДАЧИ, АРХИТЕКТУРА И БЕЗОПАСНОСТЬ

## АКТУАЛЬНОСТЬ

В современных условиях стремительной цифровизации и роста числа онлайн-сервисов управление паролями становится критически важной задачей обеспечения информационной безопасности. Среднестатистический пользователь имеет десятки учётных записей, что приводит к использованию слабых, повторяющихся паролей, легко взламываемых злоумышленниками. Согласно исследованиям, более 80% утечек данных связаны с компрометацией учётных данных. Существующие облачные решения вызывают недоверие из-за рисков централизованного хранения, сложности интерфейсов и отсутствия прозрачности криптографических механизмов. Разработка локального менеджера паролей с открытой архитектурой и современной криптографической защитой позволяет пользователям контролировать свои данные, применять стойкие алгоритмы шифрования и обеспечивать надёжное хранение паролей без передачи их третьим сторонам. Данная работа актуальна как с практической точки зрения для конечных пользователей, так и с образовательной – как пример применения криптографических технологий в реальных приложениях.

## 1.1 Задачи проекта

В рамках данной курсовой работы были поставлены и успешно решены следующие задачи:

1. **Проектирование клиент-серверной архитектуры** - разработка архитектуры с чётким разделением компонентов, выбор технологического стека (libsodium, Qt, TCP/JSON).

2. **Реализация серверной части** - разработка TCP-сервера с системой аутентификации, хранилищами паролей, механизмом восстановления через seed-фразу.

3. **Разработка клиентского приложения** - создание графического интерфейса на Qt с функциями управления паролями, генерации и оценки сложности паролей.

4. **Реализация криптографической защиты** - интеграция современных алгоритмов (Argon2id для хеширования, AES-256-GCM для шифрования хранилищ).

5. **Обеспечение безопасности данных** - защита паролей при хранении и передаче, валидация входных данных, защита от словарных атак.

6. **Контейнеризация и документирование** - подготовка Docker-образа для развёртывания, создание полной технической документации проекта.

### 1.2.2 Уровни архитектуры

**Уровень 1: Библиотеки и фреймворки**
- libsodium - криптографические примитивы
- Qt Framework - графический интерфейс
- nlohmann/json - сериализация данных
- POSIX sockets - сетевое взаимодействие

**Уровень 2: Базовые структуры данных**
- HashTableUsers - хеш-таблица пользователей
- UserHashTable - хеш-таблица паролей
- Криптографические утилиты (common_utils)

**Уровень 3: Бизнес-логика**
- AuthManager - управление аутентификацией
- PasswordGenerator - генерация паролей
- PasswordEvaluator - оценка сложности паролей

**Уровень 4: Сетевой уровень**
- Server - серверный компонент
- Client - клиентский компонент
- Протокол обмена (JSON поверх TCP)

**Уровень 5: Уровень представления**
- MainWindow - графический интерфейс пользователя
- Формы регистрации, входа, управления паролями

### 1.2.3 Протокол взаимодействия

Клиент и сервер обмениваются JSON-сообщениями через TCP-соединение. Типовые запросы:

1. **Регистрация**: `{"action": "register", "login": "...", "password": "...", "codeWord": "..."}`
2. **Вход**: `{"action": "login", "login": "...", "password": "...", "codeWord": "..."}`
3. **Синхронизация**: `{"action": "sync", "login": "...", "vault": {...}}`
4. **Восстановление**: `{"action": "recover", "login": "...", "seedPhrase": "..."}`

### 1.2.4 Хранение данных

**Серверная часть:**
- `users.dat` - файл с хеш-таблицей пользователей (учётные данные, хеши, соли)
- `vaults/<username>.vault` - зашифрованные хранилища паролей для каждого пользователя

**Клиентская часть:**
- Временное хранение в памяти после загрузки с сервера
- Локальное кеширование для работы в offline-режиме (опционально)

## 1.3 Обеспечение безопасности

Система реализует многоуровневую защиту данных, основанную на современных криптографических стандартах и лучших практиках безопасной разработки.

**Криптографическая защита:** Для хеширования паролей используется алгоритм Argon2id – память-интенсивная функция деривации ключа, защищающая от brute-force атак, rainbow tables и GPU-ускоренного перебора. Хранилища паролей шифруются с использованием AES-256-GCM (аутентифицированное шифрование), где ключ производится от кодового слова пользователя через PBKDF2. Генерация случайных данных (seed-фразы, соли, инициализационные векторы) осуществляется криптографически стойким генератором из библиотеки libsodium.

**Защита от угроз:** Модуль PasswordEvaluator анализирует пароли на предмет наличия в словаре распространённых паролей, а PasswordGenerator создаёт криптостойкие пароли с настраиваемыми параметрами. Архитектура предусматривает возможность интеграции TLS/SSL для защиты сетевого трафика. Компрометация базы паролей предотвращается шифрованием AES-256-GCM с уникальными ключами для каждого пользователя. Высокие параметры Argon2id обеспечивают защиту от атак перебора. При утере мастер-пароля предусмотрен механизм восстановления через seed-фразу из 12 слов. Чувствительные данные в памяти очищаются после использования.

**Соответствие стандартам:** Реализация соответствует российским и международным стандартам: ГОСТ Р 50922-2006 (защита информации), FIPS 197 (AES), RFC 9106 (Argon2), NIST SP 800-63B (аутентификация).

**Границы применения:** Система предназначена для безопасного хранения паролей и не заменяет многофакторную аутентификацию, системы обнаружения вторжений, антивирусное ПО или комплексные IAM-решения. Пользователям рекомендуется использовать уникальное кодовое слово, надёжно сохранить seed-фразу, регулярно обновлять мастер-пароль и не использовать систему на скомпрометированных устройствах.

## ЗАКЛЮЧЕНИЕ

В рамках данной курсовой работы была разработана полнофункциональная система управления паролями с клиент-серверной архитектурой, демонстрирующая практическое применение современных криптографических технологий.

### Достигнутые результаты

1. Реализован TCP-сервер с аутентификацией и шифрованными хранилищами паролей
2. Создан графический интерфейс на Qt с функциями генерации и оценки паролей
3. Интегрированы криптографические алгоритмы Argon2id и AES-256-GCM

Система может использоваться для безопасного хранения паролей конечными пользователями, служит образовательным примером применения криптографических протоколов и демонстрирует лучшие практики разработки безопасного ПО.
